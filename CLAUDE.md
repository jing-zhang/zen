## Project Overview

A cross-platform desktop Pomodoro-style focus timer app. Built with **Wails v2** (Go backend) + **Vue 3** (TypeScript frontend). The app lets users select focus durations (presets or custom 1-60 min), runs a countdown with arc progress visualization, and notifies on completion.

## Architecture

### Two-layer architecture (Wails binding layer)

**Go backend** (`desktop-zen-app/`) — config persistence, OS notifications, audio playback:
- `main.go` — Wails app entry point, embeds frontend dist, binds App struct
- `app.go` — App struct with LoadConfig/SaveConfig/SessionComplete/SessionCancelled methods, config file at `~/.config/desktop-zen-app/config.json`
- `assets.go` — embedded `completion.wav`, plays via beeep.Beep, sends OS notifications via beeep.Notify
- `app_test.go` — 18 tests for config CRUD, clamping, edge cases

**Vue 3 frontend** (`desktop-zen-app/frontend/`) — UI + client state:
- `src/main.js` — Vue app bootstrap with Pinia
- `src/App.vue` — root component, conditionally renders MainScreen (idle) or ZenScreen (zen mode), fallback alert modal
- `src/style.css` — CSS variables for theming (dark theme), Nunito font, global resets
- `src/stores/zenStore.ts` — **central state**: mode (idle/zen), selectedDuration, activePreset, remainingSeconds, totalSeconds, arcProgress getter, tick/startSession/completeSession/cancelSession actions
- `src/stores/languageStore.ts` — i18n (en/zh), persists to localStorage, auto-detects browser language
- `src/components/MainScreen.vue` — idle mode: TopPanel + StartButton
- `src/components/ZenScreen.vue` — active session: ArcIndicator + TimerDisplay + StartButton (as Cancel)
- `src/components/TopPanel.vue` — preset buttons, duration input, language toggle
- `src/components/StartButton.vue` — big center button (Start or Cancel depending on mode)
- `src/components/ArcIndicator.vue` — SVG circle progress ring
- `src/components/TimerDisplay.vue` — MM:SS countdown
- `src/components/PresetButton.vue` — individual preset pill button
- `src/components/DurationSelector.vue` — number input for custom duration

### Data flow

1. User sets duration (preset click or custom input) → updates Pinia `zenStore.selectedDuration`
2. User clicks Start → `startSession()` → mode='zen', sets timers, starts `setInterval` with drift correction via `Date.now()`
3. Timer ticks → `remainingSeconds` decrements → `arcProgress` computed property updates → SVG arc shrinks
4. Timer hits 0 → `completeSession()` → calls Go `SessionComplete()` (notification + audio) → mode='idle'
5. On close → `saveAndClose()` → calls Go `SaveConfig(duration)` → persisted to JSON file

### Wails bindings

Frontend imports from `../../wailsjs/go/main/App` (auto-generated by Wails):
- `LoadConfig()` → `app.LoadConfig()` Go method
- `SaveConfig(duration)` → `app.SaveConfig()` Go method
- `SessionComplete()` → `app.SessionComplete()` Go method
- `SessionCancelled()` → `app.SessionCancelled()` Go method

## Commands

All commands run from `desktop-zen-app/` unless noted.

```bash
# Run in dev mode (hot-reload frontend)
./run.sh          # or: wails dev

# Build binary
./build.sh        # or: wails build

# Run all tests
./test.sh

# Frontend tests only
cd frontend && npm run test -- --run   # 192 tests (unit + property-based)

# Go backend tests only
go test ./...                          # 18 tests

# Run single frontend test file
cd frontend && npx vitest --run test/zenStore.test.ts

# Install frontend deps
cd frontend && npm install
```

## Test Structure

- **Frontend** (`frontend/test/`): 11 files, vitest + jsdom + fast-check (property-based). Mocks Wails bindings via `vi.mock()`. Setup in `test/setup.ts`.
- **Backend** (`app_test.go`): standard Go table-driven tests, uses `t.TempDir()` for isolated config files.

## Key Design Decisions

- Duration clamped to [1, 60] on both frontend (zenStore.ts) and backend (app.go) — defense in depth
- Timer uses `Date.now()` delta for drift correction, not naive `setInterval` counting
- Config persists to platform user config dir via `os.UserConfigDir()`
- Language preference in localStorage, not backend config
- Fallback alert modal shown if OS notification permission denied
- `SessionCancelled()` is a no-op on the Go side (exists for API symmetry)
